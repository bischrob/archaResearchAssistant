Documentation

OpenAI API key: set OPENAI_API_KEY in your local .env file.

Zotero: set ZOTERO_DB_PATH, usually ZOTERO_STORAGE_ROOT, and keep ZOTERO_REQUIRE_PERSISTENT_ID=1 so ingest fails if Zotero rows are missing stable IDs.

Zotero WebDAV fallback: if local Zotero storage is not reachable, set ZOTERO_WEBDAV_URL, ZOTERO_WEBDAV_USERNAME, ZOTERO_WEBDAV_PASSWORD, and optionally ZOTERO_WEBDAV_CACHE_DIR in .env.

Linked-file mappings: if Zotero uses linked Windows or UNC paths, set ZOTERO_LINKED_PATH_MAP_JSON to map those prefixes to reachable local paths.

Zotero plugin migration step: if your library still uses linked PDFs, run Tools > RAG Sync > Normalize Linked PDFs To Stored Attachments in Zotero before relying on WebDAV-backed sync.

Qwen3: use Qwen/Qwen3-4B-Instruct-2507 as the recommended base model and set QWEN3_MODEL_PATH.

LoRA release asset: use the published adapter release and set QWEN3_CITATION_ADAPTER_PATH.

LoRA release asset

API bearer token: only enter it in the System panel if backend auth is enabled.

Recommended default: use Zotero DB (anti-join) with ingest enabled and skip-existing enabled.

Dry run: turn on Dry run to check missing counts without ingesting.

Attachment resolver order: local Zotero storage first, then cached WebDAV fallback for Zotero-managed storage: attachments, then linked-path mappings, then unresolved-path reporting.

Anystyle: build and start it with Docker if you want the recommended citation parser:

docker compose build anystyle
docker compose up -d anystyle

Recommended parser mode is CITATION_PARSER=qwen_refsplit_anystyle.

Stop behavior: Stop Sync requests cancellation. The job becomes cancelled when processing stops.

Progress meaning: early progress reports Zotero detection, then ingest progresses file-by-file.

Zotero plugin overlay: while running it shows Run in Background and Cancel Sync; after completion, failure, or cancellation it shows Close.

Citation-first lookup: use paper-scoped search when you already know the paper identity.

Count Results By = Papers returns top papers and uses Chunks per Paper for supporting snippets.

Count Results By = Chunks returns best individual chunks directly.

Use smaller result counts for faster lookup, then widen only if needed.

Zotero DB path missing: set ZOTERO_DB_PATH.

Zotero storage missing: set ZOTERO_STORAGE_ROOT.

Zotero WebDAV fallback: add the ZOTERO_WEBDAV_* variables in .env and restart the API.

Linked files still failing: add ZOTERO_LINKED_PATH_MAP_JSON if Zotero uses Windows or UNC linked paths that need to map to local Linux/WSL paths.

Best long-term fix for linked files: normalize them into Zotero-managed stored attachments from the Zotero plugin, then let Zotero sync them to WebDAV.

Anystyle not running: start the Docker service.

Qwen model or LoRA missing: set QWEN3_MODEL_PATH and QWEN3_CITATION_ADAPTER_PATH.

Bearer token mismatch: make the UI token match API_BEARER_TOKEN.

Neo4j or Docker unavailable: run preflight and restart services.
