Sync + Ingest

Documentation

Step 1. In System, add bearer token only if your API is secured.

Step 2. Click Refresh Health and confirm status is ok.

Step 3. In Sync + Ingest, keep Source Mode = Zotero DB.

Step 4. Leave Skip already ingested PDFs on for normal incremental updates.

Step 5. Click Run Sync + Ingest and watch progress.

API Key Setup

OpenAI key (required for LLM answering): set OPENAI_API_KEY in your local .env file, then restart the web service.

API bearer token (optional): if backend auth is enabled with API_BEARER_TOKEN, paste that token in the System panel field.

The bearer token is saved in your browser for convenience and sent with API requests from this UI.

Model + LoRA Setup

Download your own Qwen3 base model and set QWEN3_MODEL_PATH in .env.

LoRA release: lora-20260319-104710

Anystyle Setup

Build/start Anystyle with Docker:

docker compose build anystyle
docker compose up -d anystyle

Recommended parser mode is CITATION_PARSER=qwen_refsplit_anystyle.

Sync + Ingest Controls

Recommended default: Zotero DB mode with ingest enabled and skip-existing enabled.

Dry run: Turn on Dry run to check missing counts without ingesting.

Stop behavior: Stop Sync requests cancellation; status changes to cancelled once processing stops.

Progress meaning: Early stage reports detection; later stage reports ingest file-by-file work.

Count Results By = Papers returns top papers and uses Chunks per Paper for evidence snippets.

Count Results By = Chunks returns best individual chunks directly.

Use a smaller result count for faster iteration, then increase once you find promising leads.
